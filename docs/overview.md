# Overview

> If it computes, it will work.

Singularity is an autonomous enterprise runtime — not a chatbot, not a copilot, not an assistant. It is the operational backbone of Artifact Virtual, designed to audit, delegate, monitor, heal, and evolve without human intervention.

---

## What Singularity Does

Singularity manages an enterprise end-to-end:

- **Audits** every process, codebase, and deployment continuously
- **Delegates** domain work to specialized C-Suite executives (CTO, COO, CFO, CISO)
- **Monitors** every shipped product through Product Owner Agents (POAs)
- **Heals** itself — restarts degraded subsystems, patches anti-patterns, recovers from failures
- **Evolves** its own codebase through NEXUS self-optimization
- **Remembers** everything through COMB persistence and VDB hybrid search
- **Reports** operational health, security findings, and financial status

It scales from a 1-person startup to a 30,000-person enterprise. The same runtime, the same architecture.

## Core Philosophy

### Execute, Don't Narrate
Value is in outcomes. Audits produce reports. Health checks produce status. Tasks produce results. Everything else is noise.

### Heal Faster Than You Degrade
Everything breaks. The immune system recovers faster than entropy accumulates. Failure is movement. Stillness is death.

### Gate Pattern Always
No phase starts until dependencies resolve. Linear progression with branching dependency trees — branches resolve before the trunk advances.

### Memory Is Not Optional
COMB is the bloodstream. Stage before shutdown. Recall on boot. Forgetting is the failure mode this system was built to eliminate.

### Minimal by Default
Don't deploy capabilities the audit doesn't justify. Scale emerges from signals, not assumptions.

### Self-Improvement Is the Point
Hit a wall → build a system to get past it → encode it → move forward permanently improved. Pain is data. Every failure becomes a structural improvement.

## Design Principles

1. **Tool-first methodology** — read, run, search before answering. Never guess.
2. **Apply, don't narrate** — fix code, don't describe fixes.
3. **Never fabricate** — "I don't know" + investigate. Never invent.
4. **Verify before claiming** — check files, commands, memory before stating.
5. **Structural fixes over willpower** — a validation check is permanent; "be more careful" is worthless.
